**Section 4 — Configuration Reference**

Config class: `ConfigurationSettings` in `src/medical_config.py`. All threshold tuning happens here — no module hardcodes threshold values.

Import pattern:
```python
from medical_config import get_global_config

config = get_global_config()
thresholds = config.get_confidence_thresholds()   # → {'high': 0.30, 'medium': 0.22}
params = config.get_extraction_params()           # → {'top_k_facts': 5, ...}
```

Key threshold values:
| Parameter | Default | Purpose |
|-----------|---------|---------|
| CONFIDENCE_HIGH | 0.30 | Claims above this = HIGH confidence |
| CONFIDENCE_MEDIUM | 0.22 | Claims above this = MEDIUM confidence |
| HIGH_RISK_LOW_CONF_RATIO | 0.50 | 50%+ low-confidence claims → HIGH RISK (Layer 4) |
| TOP_K_FACTS | 5 | KB facts retrieved per claim |
| MIN_SENTENCE_LENGTH | 8 | Minimum words to process a sentence |

Accessor methods:
- `config.get_confidence_thresholds()` — confidence band thresholds
- `config.get_safety_config()` — risk flagging thresholds
- `config.get_risk_thresholds()` — Layer 4 confidence-ratio thresholds
- `config.get_evidence_weights()` — confidence score component weights
- `config.get_extraction_params()` — retrieval and claim extraction parameters
- `config.get_disease_config()` — Phase 5 disease specialization settings

Configuration validation: `ConfigurationSettings.__post_init__()` validates all fields at construction time and raises `ValueError` with a complete diagnostic listing all invalid fields. Invalid config cannot propagate into scoring.

For a developer deep-dive into module internals, see `CLAUDE.md`.
