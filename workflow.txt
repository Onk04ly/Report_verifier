# Medical AI Hallucination Detection — Execution Workflow

Run phases in order. Each phase produces artifacts consumed by the next.

## Phase 1 — Data Collection

**Goal:** Build the raw medical knowledge base from PubMed.

```bash
python src/pubmed_fetcher.py
```

**Output:** `data/expanded_knowledge_base.csv` (raw KB, 500+ articles)

**Notes:**
- Set `NCBI_API_KEY` environment variable for 10 req/s instead of 3 req/s.
- Fetches 130+ medical topics with year/journal segmentation to work around NCBI's 10 000-result cap.

---

## Phase 2 — KB Preprocessing and Embedding

**Goal:** Clean, enhance, and embed the knowledge base for runtime retrieval.

```bash
python src/medical_preprocessor.py
```

**Output:**
- `data/expanded_knowledge_base_preprocessed.csv` — quality-graded, NER-annotated KB
- `data/kb_embeddings_preprocessed.npy` — pre-computed SciBERT embeddings (regenerating is slow)

**Notes:**
- Runs offline once. Do not re-run unless the raw KB changes.
- Generates SciBERT embeddings; requires GPU or patience on CPU.

---

## Phase 3 — Claim Extraction (development/test)

**Goal:** Test claim extraction and verification engine in isolation.

```bash
python src/claim_extractor_fixed.py
```

**Output:** Claim extraction results printed to stdout (does not write output files).

**Notes:**
- Runs against the preprocessed KB. Useful during development to verify FAISS index and confidence scoring.

---

## Phase 4 — Full Verification Pipeline

**Goal:** Run the complete hallucination detection pipeline on medical summaries.

```bash
python src/medical_verifier.py
```

**Output:**
- `outputs/medical_verification.json` — structured verification results with risk assessment
- `outputs/medical_verification.csv` — tabular claims export
- `outputs/medical_verification_safety_log.json` — safety validation log

**Notes:**
- Applies 4-layer risk model (medical plausibility → severity count → implausibility → confidence ratio).
- The Responsible AI layer (safety warnings, expert-review flags, disclaimer) is integrated into this phase — it is not a separate step.

---

## Phase 5 — Interactive Analysis

**Goal:** Explore verification results interactively.

```bash
jupyter notebook medical_verification_dashboard.ipynb
```

**Output:** Interactive claim/risk visualization in the browser.

---

## Phase 6 — Report Generation

**Goal:** Generate an HTML report from verification results.

```bash
python src/report_generator.py
```

**Output:** Self-contained HTML report built from `outputs/medical_verification.json`.

---
